Sketch Modeling

3D mesh generation from a sketch

I have designed and developed SketchModeling, a generative model that transforms 2D sketches into 3D models.

Recognizing the need for an AI-assisted tool that can directly generate model drafts from sketches, I propose a solution that integrates the "sketch to image" and "image to 3D" processes. This integration aims to minimize the inconvenience caused by model conversion and interfacing, thereby enhancing the efficiency of AI tools and assisting designers and artists in utilizing AI more effectively.

Design

In conceptualizing the transformation from sketch to 3D model, I devised a linear workflow that combines "sketch to image" and "image to 3D" processes. After reviewing relevant literature, I finalized the workflow for SketchModeling, as shown below

SketchModeling consists of three subprocesses and five models, with the complete workflow as follows:

1. The user inputs a sketch, a prompt, and several optional parameters.

2. StableDiffusion and ControlNet generate an image that aligns with the prompt description and sketch shape.

3. RMBG processes the image to produce a version with the main subject extracted and a blank background.

4. InstantMesh generates a 3D model in .obj or .glb format based on the processed image.

5. InstantMesh involves two steps: first, using the Zero123++ model to create a multi-view image from various perspectives, and second, generating a 3D model from this multi-view image.

Development

Model Utilization

I primarily utilized the Diffusers library for this part of the development.

Hugging Face's Diffusers library offers a simple inference solution and is an efficient toolkit for using and building diffusion models. Its core components include models and schedulers. The Pipeline class in Diffusers bundles these components for convenience, significantly reducing manual configuration time.

All required models in this project are invoked through pipelines. During development, I pre-constructed all necessary pipelines and downloaded all models before loading the Gradio interface to prevent long loading times that could hinder user experience

Frontend Development

I used the Gradio library for frontend development. Gradio makes it extremely convenient to build interfaces for AI-driven apps, allowing us to implement all frontend and backend functionalities using Python code without writing HTML, CSS, or JavaScript.
